# REQ-002 阶段流转、风险与归档

> 来源场景：`SCN-REQ-002`
>
> 状态：DRAFT / 待联合评审

## 用户目标

通过已批准的跨系统事件减少人工拖卡，同时保证人工回退、阶段历史、风险处置和自动归档可追踪。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-REQ-006 | Must | SIT 准入通过进入“SIT中”，UAT 准出通过进入“待上线”，上线完成进入“已上线”；重复事件不得重复流转。 |
| FR-REQ-009 | Must | 人工流转校验权限、目标和当前状态；回退仅到相邻前序阶段且必须填写原因。 |
| FR-REQ-010 | Must | 每次流转关闭当前阶段区间、创建目标阶段区间并保留全部进入、离开和回退历史。 |
| FR-REQ-011 | Must | 上线满 90 个自然日的需求由 J13 自动归档，归档后默认只读且不进入在建列表。 |
| FR-REQ-017 | Must | 风险记录类型、描述、严重度、责任人、状态和解决时间，并纳入受阻统计。 |
| FR-CROSS-001 | Must | 缺业务标识、对象不存在、状态非法或载荷失败时不得静默丢弃或部分提交，须形成失败/冲突记录。 |
| FR-CROSS-002 | Must | 可重试集成操作使用幂等键；有限重试耗尽后转人工，不得无限重试。 |

## 接口、数据与作业

- 接口：`IF-006`、`IF-008`～`IF-013`、`IF-100`～`IF-103`。
- 数据：`DATA-REQ`、`DATA-SYNC`。
- 作业：`J01`、`J10`、`J11`、`J13`。
- 外部系统：知脉、知测。

## 非范围

- 未经批准增加其他自动拖卡规则。
- 覆盖或删除历史阶段区间、风险或操作日志。
- 在本需求中定义重试次数、退避和死信参数。
