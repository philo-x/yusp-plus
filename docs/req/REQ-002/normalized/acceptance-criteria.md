# REQ-002 验收标准

## AC-REQ-002：自动推进与人工回退

- **Given** 需求处于合法/非法前置状态，事件包含首次、重复、乱序和版本变化。
- **When** 接收三类批准事件并执行合法相邻回退、跨级回退、无原因回退和越权回退。
- **Then** 合法事件只流转一次；异常事件进入冲突记录；只有授权且有原因的相邻回退成功；当前阶段唯一且历史可还原。

## AC-REQ-003-ARCHIVE：90 日归档

- **Given** 上线第 89、90、91 天需求。
- **When** 首次及重复执行 J13。
- **Then** 边界日前不归档，第 90 天合格调度后归档；重复执行不重复记账；归档需求默认只读并退出在建视图。

## AC-REQ-005：风险处置

- **Given** 不同权限的创建者、责任人和管理者。
- **When** 创建、解决和关闭不同严重度风险。
- **Then** 必填字段、权限、状态和日志正确；未关闭计数与横幅及明细一致。

## AC-INT-001（REQ-002 子集）：集成失败

- 缺标识、不存在、状态冲突和非法载荷不得部分提交。
- 重复事件不产生重复结果。
- 暂时失败有限重试，耗尽后状态明确且可人工处置。
