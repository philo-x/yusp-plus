# REQ-002 需求管理 - 需求详情与列表筛选 - 验收标准说明书 (acceptance-criteria.md)

> **任务包编号**：REQ-002  
> **任务包名称**：需求管理 - 需求详情与列表筛选  

---

## 验收标准列表 (Acceptance Criteria)

- **`AC-036`**：**项目经理唯一性拦截测试**：在编辑需求详情时尝试添加第二个项目经理，前端校验弹框阻断，API 抛出 HTTP 400 校验错误。
- **`AC-037`**：**上线止计时测试**：将某需求通过自动拖卡更新至“已上线”，24 小时后查询该需求的“当前状态停留天数”，数值保持上线瞬间的天数，不再增加。
- **`AC-038`**：**回退恢复计时测试**：将已上线需求回退至“待上线”，系统次日重新扫表，停留天数与创建天数根据停滞时间恢复累计。
- **`AC-039`**：**复合条件筛选测试**：构建“所属部落 = 零售部落 AND (需求类型 = 紧急 OR 停留天数 > 5)”组合条件筛选，列表返回数据精确满足逻辑判定。
- **`AC-040`**：**自定义显示列持久化测试**：用户在列表隐藏“风险问题数”列并刷新页面，该列持续隐藏；退出账号重新登录，该列仍然保持隐藏状态。
