# REQ-002 需求管理 - 需求详情与列表筛选 - 业务规则说明书 (business-rules.md)

> **任务包编号**：REQ-002  
> **任务包名称**：需求管理 - 需求详情与列表筛选  

---

## 业务规则列表 (Business Rules)

- **`BR-026`**：**项目经理唯一性校验**：每个需求记录在知效平台中必须且仅能绑定唯一一名项目经理，修改项目经理必须留痕。
- **`BR-027`**：**计时中断与恢复规则**：
  1. 需求状态变更进入“已上线”后，当前状态停留天数、受理天数和创建天数即刻**停止累计**。
  2. 需求从“已上线”或后续阶段发生状态回滚时，相关计时器**自动恢复累计**。
  3. 后台表必须完整保留需求在每个历史阶段的实际停留天数（非简单覆盖）。
- **`BR-028`**：**高级复合筛选逻辑**：筛选条件组之间支持 AND（且）与 OR（或）逻辑嵌套；当且仅当组内与组间布尔式评估为 true 时方返回对应记录。
- **`BR-029`**：**导出数据一致性约束**：导出 Excel/CSV 生成的数据集合，必须与当前页面点击“导出”时 UI 显示的过滤条件和列选择完全一致，禁止导出越权数据。
- **`BR-030`**：**个性化列配置持久化**：用户设置的默认显示列参数必须绑定其 `user_id` 保存在 `user_column_config` 表中，跨终端登录自动生效。
