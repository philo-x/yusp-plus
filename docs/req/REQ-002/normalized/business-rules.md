# REQ-002 业务规则

| ID | 规则 |
|---|---|
| RULE-REQ-001 | 每个有效需求任一时点只有一个当前阶段；阶段历史不得覆盖。 |
| RULE-REQ-002 | 当前已确认自动拖卡仅限 SIT 准入、UAT 准出和上线完成三类事件。 |
| REQ-002-RULE-001 | 外部事件按租户+源系统+事件 ID+方向幂等，并保存源版本和载荷摘要。 |
| REQ-002-RULE-002 | 乱序或与当前状态不兼容的事件进入异常，不得强制改写当前阶段。 |
| REQ-002-RULE-003 | 回退重入相同阶段必须创建新阶段实例，各独立区间均可还原。 |
| REQ-002-RULE-004 | 未关闭风险计入受阻统计；创建、解决和关闭必须留痕。 |
| REQ-002-RULE-005 | 上线满 90 个自然日以业务时区计算；边界日前不归档，首次合格调度后归档。 |
| DATA-RULE-001 | 同步对象保存源系统、源标识、源版本和事件标识；对象唯一且事件幂等。 |
| DATA-RULE-002 | 当前阶段唯一；回退重入创建新实例，阶段、计划和操作历史不得覆盖。 |

## 异常与边界

- 重复事件：返回稳定结果，不新增阶段或业务结果。
- 对象不存在、缺标识、非法状态：保存失败/冲突记录并允许授权处理。
- 无权、跨级或无原因回退：拒绝且当前状态不变。
- 同一阶段多次进入：保留每个独立区间，不合并覆盖。
- 外部依赖暂时不可用：有限重试，耗尽后转人工。

| 源 ID | 规范化边界 |
|---|---|
| ERR-SYNC-001 | 缺标识、不存在或状态冲突形成失败/冲突记录，禁止静默丢弃和部分提交。 |
| ERR-SYNC-002 | 暂时不可用保留待重试状态，耗尽后转人工，禁止无限重试。 |
| EDGE-REQ-001 | 重复/乱序事件按标识和版本幂等，非法顺序进入异常。 |
| EDGE-REQ-002 | 多次回退并重入相同阶段时保存和累计每个独立区间。 |
