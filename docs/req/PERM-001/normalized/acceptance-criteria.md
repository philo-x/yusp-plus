# PERM-001 角色与数据权限管理 - 验收标准说明书 (acceptance-criteria.md)

> **任务包编号**：PERM-001  
> **任务包名称**：角色与数据权限管理  

---

## 验收标准列表 (Acceptance Criteria)

- **`AC-001`**：**管理层组织切换验证**：以部落长身份登录，需求主页横幅默认选择其主部落；下拉框选择下属小队或同等授权部落，列表和顶部横幅数值即时（<500ms）更新为所选组织数据。
- **`AC-002`**：**执行层权限隔离验证**：以开发人员身份登录，需求主页不显示部落/小队切换下拉控件；需求列表仅能检索到本人作为主办人、辅办人或参与人的需求项。
- **`AC-003`**：**报工查看菜单鉴权测试**：使用外包人员账号登录，导航菜单中隐蔽“报工查看”节点，直接调用 REST API `POST /api/workhour/view` 返回 `retCode: 40003` (越权访问)。
- **`AC-004`**：**行方负责人报工下钻测试**：行方负责人登录后打开“报工查看”，查询结果仅包含该负责人名下绑定的外包员工，不包含其他行方负责人名下员工。
- **`AC-005`**：**自定义字段权限边界验证**：项目经理尝试在全局作用域下新增自定义字段，系统弹出权限拒绝提示，保存接口校验拦截。
