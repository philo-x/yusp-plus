# REQ-001 需求价值流视图与扩展字段

> 来源场景：`SCN-REQ-001`
>
> 状态：DRAFT / 待联合评审

## 用户目标

授权用户从同一需求集合中通过泳道、列表、甘特图和横幅理解阶段、计划、超期情况，并按个人需要筛选和配置显示。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-REQ-001 | Must | 接收知脉需求主数据，并以源业务标识维护唯一、可追踪映射。 |
| FR-REQ-002 | Must | 提供泳道、列表和甘特图，切换时保持当前权限和筛选范围。 |
| FR-REQ-003 | Must | 泳道按 13 个标准阶段展示，每个有效需求只显示在一个当前阶段。 |
| FR-REQ-004 | Must | 列表展示编号、名称、唯一 PM、阶段、计划上线时间、类型、风险数、阶段停留、受理和创建天数。 |
| FR-REQ-005 | Must | 用户可配置列项、顺序和宽度，配置按用户持久化且互不影响。 |
| FR-REQ-007 | Must | 支持文本、枚举、日期、数值的 AND/OR 嵌套筛选，并可保存个人筛选。 |
| FR-REQ-008 | Must | 甘特图按日/周/月展示计划与实际时间，并区分研发、SIT、UAT、已上线和超期。 |
| FR-REQ-012 | Must | 横幅展示权限范围内需求总数、超期/受阻数、在建紧急数和组织大盘，前三项可钻取。 |
| FR-REQ-013 | Must | 组织视角只能在授权范围内切换，执行层不得扩大数据范围。 |
| FR-REQ-014 | Must | 支持全局、部落、小队、项目组四级自定义字段和六类批准字段类型。 |
| FR-REQ-015 | Must | 全局有效字段最多 20 个，每个项目私有有效字段最多 10 个。 |
| FR-REQ-016 | Must | 自定义字段仅存于知效，不向外同步；导入导出包含当前作用域有效字段。 |

## 数据与接口

- 数据：`DATA-REQ`、`DATA-ORG`。
- 接口：`IF-001`～`IF-005`、`IF-018`～`IF-027`、`IF-100`、`IF-104`。
- 外部依赖：知脉需求主数据、组织与人员权限数据。

## 非范围

- 在知效修改知脉主数据的权威字段。
- 将自定义字段通过 MQ、中间库或外部接口发送。
- 本需求不负责阶段自动推进、回退、风险处置和归档，它们归入 `REQ-002`。
