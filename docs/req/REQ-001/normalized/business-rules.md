# REQ-001 业务规则

| ID | 规则 |
|---|---|
| RULE-REQ-001 | 有效需求任一时点只能有一个当前阶段，三种视图必须读取同一权限与筛选真值集。 |
| RULE-REQ-003 | 全局有效自定义字段最多 20 个，每个项目私有有效字段最多 10 个，且不得向外部系统同步。 |
| REQ-001-RULE-001 | 同租户+知脉源标识唯一；重复或更新事件不得创建第二个业务需求。 |
| REQ-001-RULE-002 | 个人筛选和显示配置仅对本人有效，不得改变其他用户或默认全局配置。 |
| REQ-001-RULE-003 | 归档需求默认不进入在建视图；缺少计划时间的需求可展示，但甘特图必须标记数据不完整。 |
| REQ-001-RULE-004 | 横幅聚合和钻取必须使用相同口径、权限、组织与筛选条件。 |
| REQ-001-RULE-005 | 自定义字段值须与字段类型匹配；数值和日期筛选不得使用文本或 JSON 比较。 |
| REQ-001-RULE-006 | 字段配额须在作用域内原子占用，归档字段保留历史值但不能新增有效值。 |
| DATA-RULE-003 | 项目、参与人、系统归属、字段可见对象和个人筛选须以关系/配置表达，字段配额原子占用。 |
| DATA-RULE-004 | 每对象每字段仅一个有效值，值类型必须匹配，数值/日期不得依赖 JSON 或文本比较。 |

## 异常与边界

- 非法筛选字段或操作符：拒绝查询并指出具体条件。
- 越权组织：不出现在可选项中，直接接口请求同样拒绝。
- 来源需求重复：幂等更新映射并保留事件日志。
- 并发创建第 10/11 或 20/21 个字段：只有配额内请求成功，其他请求失败且不污染已有数据。
