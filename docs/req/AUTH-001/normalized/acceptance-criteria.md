# AUTH-001 验收标准

## AC-AUTH-001：统一认证生命周期

- **Given** 有效、无效、过期和被注销的知悉令牌。
- **When** 分别执行登录、回调、刷新、注销和失效后访问。
- **Then** 有效用户建立会话；无效或过期请求被拒绝并可重新认证；日志和响应不暴露令牌或敏感配置。

## AC-AUTH-002：角色与数据作用域

- **Given** 角色×组织×业务对象的允许/拒绝真值矩阵。
- **When** 用户从页面和直接接口查询或修改本人、授权对象和越权对象。
- **Then** 所有允许项成功、拒绝项失败，页面与接口结果一致，组织切换不扩大作用域。

## AC-AUTH-003：人员映射异常

- **Given** 统一身份有效但不存在有效人员/组织映射的用户。
- **When** 用户登录并请求业务数据。
- **Then** 不授予业务数据访问权，产生可查询异常且管理员可定位原因。

## 证据与通过条件

- 覆盖 `NFR-SEC-001`、`NFR-SEC-002` 和 `IF-082`～`IF-092`。
- 记录环境、版本、执行人、日期、真值矩阵和证据编号。
- `TBD-SEC-001` 未关闭前，多角色和兼职场景不得最终通过。
