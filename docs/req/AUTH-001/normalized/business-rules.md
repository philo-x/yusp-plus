# AUTH-001 业务规则

| ID | 规则 |
|---|---|
| AUTH-001-RULE-001 | 除 SSO 发起/回调和批准的受信系统回调外，所有接口均须认证。 |
| AUTH-001-RULE-002 | 服务端从受信会话解析用户和租户，不接受客户端任意指定其他用户、租户或组织作用域。 |
| AUTH-001-RULE-003 | 查询按数据作用域裁剪；写操作同时校验功能权限、对象范围和当前状态。 |
| AUTH-001-RULE-004 | 多角色或跨组织权限冲突在规则批准前按最小权限处理，特权操作必须审计。 |
| AUTH-001-RULE-005 | 无有效人员映射的身份只能看到可处理的无权状态，不得获得默认业务权限。 |
| AUTH-001-RULE-006 | 认证、授权和安全错误不得泄露令牌、凭据、堆栈或敏感配置。 |

## 异常和边界

| 场景 | 期望行为 |
|---|---|
| 令牌无效、过期或回调校验失败 | 拒绝登录或操作，引导重新认证并记录安全事件。 |
| 越权访问 | 返回明确的无权限结果并记录必要审计，不以空泛系统错误替代。 |
| 用户存在多个角色或跨组织兼职 | 按 `TBD-SEC-001` 批准的合并、冲突和有效期规则计算。 |
| 组织主数据暂时不可用 | 保持最近批准的有效权限或拒绝新增授权，具体恢复策略由接口运行方案批准。 |

| 源 ID | 规范化边界 |
|---|---|
| ERR-AUTH-001 | 登录失效或越权时拒绝、返回可理解错误并记录安全事件；不得只隐藏页面。 |
