# AUTH-001 统一认证与权限视角

> 来源场景：`SCN-AUTH-001`
>
> 状态：DRAFT / 待联合评审

## 目标

所有交互用户通过知悉统一认证进入知效平台，并且只能访问其角色、组织和业务关联共同允许的菜单、指标、对象和操作。

## 参与者

- 所有交互用户
- 知悉统一认证平台
- 组织与人员主数据提供方
- 安全管理员、平台管理员

## 前置条件

- 用户在知悉中有效。
- 用户、组织、角色和业务关联已同步，或同步异常可被识别和处理。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| NFR-SEC-001 | Must | 系统必须使用知悉 OAuth2 统一认证，支持登录、回调、刷新、注销和失效令牌处理，不得维护独立硬编码密码。 |
| NFR-SEC-002 | Must | 服务端必须对每个接口执行功能权限与数据作用域授权并遵循最小权限；仅隐藏前端入口不构成授权。 |
| AUTH-001-REQ-001 | Must | 认证成功后，系统必须加载用户的角色、组织和业务关联，建立会话并生成可追踪的权限上下文。 |
| AUTH-001-REQ-002 | Must | 用户切换页面、组织或对象时，服务端必须重新校验作用域；越权对象不得因直接调用接口而暴露。 |
| AUTH-001-REQ-003 | Must | 用户无法映射到有效人员或组织时，不得授予业务数据访问权，并必须生成可处理的同步异常。 |

## 接口与数据边界

- 接口：`IF-082`～`IF-092`。
- 数据：`DATA-ORG`、`DATA-SYNC`。
- 外部依赖：知悉 OAuth2、组织和人员主数据。

## 非范围

- 独立用户名密码、匿名访问和公众互联网访问。
- 在知效中替代统一身份或组织主数据职责。
- 在本需求中自行决定多角色、兼职或临时授权优先级。
