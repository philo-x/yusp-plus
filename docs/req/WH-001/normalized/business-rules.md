# WH-001 业务规则

| ID | 规则 |
|---|---|
| RULE-TASK-002 | “进行中”即点亮，离开即取消；每人每天最多 5 个。 |
| RULE-WH-001 | 可维护日期早于当天且在向前 14 个自然日（含）内。 |
| RULE-WH-002 | 同人同日有效报工总分钟不得超过最新有效考勤。 |
| RULE-WH-003 | 人力外包无点亮任务时生成任务为空的异常报工。 |
| RULE-WH-004 | 已确认、已锁定或超窗口报工不得修改删除。 |
| WH-001-RULE-001 | 每人每日每任务只有一条点亮事实；J03 读取前一日最终历史态。 |
| WH-001-RULE-002 | 默认报工按人员+日期+任务幂等，总分钟必须守恒。 |
| WH-001-RULE-003 | `workload_confirm_status` 是知脉封板快照，不是知效本地审批。 |
| DATA-RULE-006 | 报工关联用户/日期和合格任务；幂等键唯一，同日总分钟受最新考勤版本约束。 |
| DATA-RULE-007 | 封板、锁定或超窗口报工不可修改删除，失败尝试也须留痕。 |

## 边界与异常

- 第 14 日允许；当天和第 15 日拒绝。
- 第 6 个点亮被拒绝，原 5 个状态不变。
- 分钟不能整除时总数仍须守恒，顺序由 `TBD-WH-001` 决定。
- 补卡缩短考勤时更新考勤并重算或转异常，不能覆盖锁定记录。
- 人员映射失败进入异常并告警，不生成无主正常报工。
- 超考勤、窗口、任务资格的变更必须拒绝并保持原数据。

| 源 ID | 规范化边界 |
|---|---|
| ERR-WH-001 | 超考勤、窗口或资格时拒绝并保持原数据，不截断分钟或替换任务。 |
| ERR-WH-002 | 人员无法映射时进入异常并告警，不生成无主正常报工。 |
| EDGE-WH-001 | 第 14 日允许；当天和第 15 日拒绝。 |
| EDGE-WH-002 | 分钟不能整除时总数守恒，尾差顺序待批准。 |
| EDGE-WH-003 | 补卡触发重算/异常，不覆盖锁定记录。 |
| EDGE-WH-004 | J03/J04 时序未确认前不得假设默认报工进入当日 07:00 批次。 |
