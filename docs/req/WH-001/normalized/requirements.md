# WH-001 点亮、考勤与个人报工

> 来源场景：`SCN-WH-001`
>
> 状态：DRAFT / 待联合评审

## 用户目标

外包员工基于真实考勤和前一日实际进行中的任务获得默认报工，并在允许窗口内核对和调整。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-TASK-009 | Must | 子任务进入“进行中”即当日点亮，离开即取消。 |
| FR-TASK-010 | Must | 每人每天最多同时点亮 5 个可报工任务，第 6 个操作被拒绝且原状态不变。 |
| FR-WH-001 | Must | J02 每日 03:00—06:00 同步外包人员近 14 天（含）考勤并按工号匹配人员。 |
| FR-WH-002 | Must | 补卡/请假变化须更新对应考勤并触发重算或异常，不覆盖已锁定报工。 |
| FR-WH-003 | Must | J03 在考勤同步成功后、08:00 前按前一日最终点亮事实生成默认报工。 |
| FR-WH-004 | Must | 有 N 个点亮任务时生成 N 条报工，总分钟严格等于考勤分钟。 |
| FR-WH-005 | Must | 人力外包无点亮任务且考勤大于 0 时生成一条任务为空的异常报工。 |
| FR-WH-006 | Must | 默认工时以人员+日期+任务为幂等键，重跑不增加记录或分钟。 |
| FR-WH-007 | Must | 外包人员可用列表和日历查看本人报工，报工页不得修改点亮任务。 |
| FR-WH-008 | Must | 可维护日期必须早于当天且位于向前 14 个自然日（含）内。 |
| FR-WH-009 | Must | 同一人员同日有效报工合计不得超过最新有效考勤分钟。 |
| FR-WH-010 | Must | 报工只关联已分配、同系统、工作量未确认的开发任务；确认后记录锁定。 |
| FR-WH-011 | Must | 新增、修改、删除、自动重算和失败尝试记录完整审计；锁定或超窗口记录不得变更。 |

## 接口、数据与作业

- 接口：`IF-032`、`IF-036`、`IF-052`～`IF-061`。
- 数据：`DATA-TASK`、`DATA-WH`。
- 作业：`J02`、`J03`。
- 外部系统：考勤平台。

## 非范围

- 在报工页面修改任务点亮状态。
- 将事务类任务用于报工。
- 在未批准算法前自行决定分钟尾差顺序或 J03/J04 时序。
