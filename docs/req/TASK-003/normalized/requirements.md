# TASK-003 任务管理 - 个人子任务与打卡点亮 - 需求说明书 (requirements.md)

> **任务包编号**：TASK-003  
> **任务包名称**：任务管理 - 个人子任务与打卡点亮  
> **关联 PRD 章节**：5.7  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-101`**：系统必须支持研发人员在任务列表中维护个人子任务的状态，当研发人员将个人子任务的状态置为“进行中”（IN_PROGRESS）时，系统视其为当日完成打卡点亮。
- **`FR-102`**：系统必须对每人每日处于“进行中”状态的个人子任务数量施加硬性上限约束：**每人每日处于“进行中”的个人子任务不得超过 5 个**。
- **`FR-103`**：当研发人员将个人子任务的状态从“进行中”修改为“已完成”、“暂停”或其他非“进行中”状态时，系统必须自动取消该子任务的当日点亮状态。
- **`FR-104`**：系统必须在报工管理页面联动展示外包人员的当日点亮任务，但报工页面绝对禁止直接修改点亮状态，必须提示“如需调整每日点亮任务，请在任务列表功能模块，将对应子任务置为‘进行中’”。
- **`FR-105`**：系统必须保证个人子任务的点亮资格与报工资格相互独立隔离：默认报工工时仅根据已点亮的开发任务生成，安全内容不作为独立任务进行点亮或报工。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-011`**：个人子任务点亮状态变更时，校验并发锁（Redisson Key `zx:task:lighting:lock:{user_id}`）并进行 Redis 缓存刷新，接口响应整体小于 100ms。
