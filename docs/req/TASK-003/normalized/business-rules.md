# TASK-003 任务管理 - 个人子任务与打卡点亮 - 业务规则说明书 (business-rules.md)

> **任务包编号**：TASK-003  
> **任务包名称**：任务管理 - 个人子任务与打卡点亮  

---

## 业务规则列表 (Business Rules)

- **`BR-106`**：**单日 5 个点亮上限硬约束**：
  - 用户尝试将第 6 个个人子任务置为“进行中”状态时，系统抛出 HTTP 200 / `retCode: 40025` 业务拦截异常，提示“突破单日最多点亮 5 个任务限制”。
- **`BR-107`**：**点亮范围限定规则**：点亮规则全量覆盖需求类开发任务和 SIT 测试任务下的个人子任务。项目类与事务类任务是否适用受待确认事项 `Q-18` 约束。
- **`BR-108`**：**报工页面只读提示约束**：报工管理界面仅允许以 Read-Only 浮框或 Tag 形式读取当日点亮任务，禁止在报工界面抛出 Toggle 开关直接修改子任务状态。
- **`BR-109`**：**安全内容与工时隔离**：安全内容附属于开发任务，不单独计算点亮次数，也不参与默认工时的拆分计算。
