# REQ-003 需求管理 - 固定横幅与指标统计 - 需求说明书 (requirements.md)

> **任务包编号**：REQ-003  
> **任务包名称**：需求管理 - 固定横幅与指标统计  
> **关联 PRD 章节**：4.6  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-036`**：系统必须在需求管理主页顶部提供固定横幅（Banner），在需求泳道图、需求列表和甘特图切换时均置顶显示且不消失。
- **`FR-037`**：固定横幅必须展示当前权限范围内的 3 项明确指标：
  1. **总需求数**：当前权限范围内在建需求数与已上线需求数之和。
  2. **超期/受阻需求数**：阶段停留超时或存在阻塞标记的风险需求数，数值必须**红色加粗**显示。
  3. **在建紧急需求数**：当前尚未上线的紧急需求数，支持点击触发交互。
- **`FR-038`**：系统必须支持横幅指标与下方需求的点击联动：
  - 点击“在建紧急需求数”，下方的泳道图/列表自动筛选仅展示在建紧急需求。
  - 点击“超期/受阻需求数”，下方的泳道图/列表自动筛选仅展示风险需求。
- **`FR-039`**：当下方需求列表应用了筛选条件时，顶部固定横幅的对应过滤区域必须高亮提示，防止用户产生误解。
- **`FR-040`**：系统必须支持管理层通过横幅切换部落或下属小队，切换后固定横幅内的统计数字与下方需求列表同步实时刷新。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-005`**：横幅指标统计接口必须使用 Redis 缓存进行聚合预计算，页面加载时横幅渲染延迟不超过 150ms。
