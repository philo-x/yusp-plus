# WH-002 业务规则

| ID | 规则 |
|---|---|
| RULE-ROL-001 | 灰度期按生效规则选择知效或知微为人员+日期的唯一权威来源，另一来源不得重复入账。 |
| WH-002-RULE-001 | 管理查询以批准的负责人、中心、小队、部落、部门或全行作用域为边界。 |
| WH-002-RULE-002 | 组织调整不得改写历史报工当时的归属快照。 |
| WH-002-RULE-003 | J04 每条同步保存状态、追踪号、尝试次数和最后错误；失败不得标记成功。 |
| WH-002-RULE-004 | 来源裁决保留候选摘要、规则版本、采用来源、明细数和分钟合计。 |
| WH-002-RULE-005 | 权威来源下所有合法任务明细均保留，不得因去重粒度误删。 |

## 异常与边界

- 越权查询或导出：服务端拒绝。
- 知脉超时：保持待重试，不标记成功。
- 重复写回：返回稳定结果，不重复核算。
- 批量部分失败：返回准确汇总和逐项原因。
- 同日多任务跨来源：先裁决来源，再保留该来源完整明细。
