# WH-002 报工管理 - 我的报工与填报限制 - 业务规则说明书 (business-rules.md)

> **任务包编号**：WH-002  
> **任务包名称**：报工管理 - 我的报工与填报限制  

---

## 业务规则列表 (Business Rules)

- **`BR-126`**：**14 天时间窗口硬性卡死**：
  - 填报/修改/删除逻辑：`work_date >= (current_date - 14days) AND work_date < current_date`。
  - 超过 14 天的超期报工，接口直接返回 HTTP 200 / `retCode: 40015`（超出 14 天报工窗口）。
- **`BR-127`**：**考勤工时天花板硬校验**：
  - 单人单日报工分钟数之和：`SUM(minutes) <= attendance_minutes`。
  - 超过考勤上限时拒绝保存，错误码 `40015`。
- **`BR-128`**：**知脉工作量确认即锁定规则**：
  - 当知脉端 `workload_confirm_status = 1` 时，对应的报工记录状态置为 `LOCKED`。
  - 任何编辑或删除请求直接拦截，返回 HTTP 200 / `retCode: 40018`（知脉工作量已封板锁定）。
- **`BR-129`**：**归属系统一致性约束**：分配开发任务与关联报工时，人员 `belong_system` 必须等于任务 `system_id`，禁止跨系统越权填报。
- **`BR-130`**：**只读派生字段防护**：需求名称、工号、公司、系统等字段在前端填报时为只读显示，底层 DAO 在保存时自动从 Task & Requirement 级联数据补充，拒绝前端传参覆盖。
