# WH-002 管理报工与知脉同步

> 来源场景：`SCN-WH-002`
>
> 状态：DRAFT / 待联合评审

## 用户目标

管理者在正确数据范围内核查外包报工，J04 将合格数据可追踪地写回知脉，并在双系统期间避免重复核算。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-WH-012 | Must | J04 每日 07:00 将符合核算条件的报工同步知脉，每条结果可追踪并按批准策略重试。 |
| FR-WH-013 | Must | 行方负责人、中心经理、团队长、小队长、PMO、部落长、部门/行领导按批准作用域查看报工。 |
| FR-CROSS-001 | Must | 同步缺标识、状态冲突、载荷失败或对象不存在须形成失败/冲突记录。 |
| FR-CROSS-002 | Must | 写回使用幂等键，有限重试耗尽后转人工。 |
| FR-CROSS-003 | Must | 批量同步/查询返回总数、成功数、失败数和逐项原因。 |
| ROL-002 | Must | 灰度期以人员+报工日期裁决唯一权威来源，并保留该来源全部合法任务明细。 |

## 接口、数据与作业

- 接口：`IF-062`～`IF-064`、`IF-100`～`IF-102`、`IF-106`。
- 数据：`DATA-WH`、`DATA-ORG`、`DATA-SYNC`。
- 作业：`J04`、`J05`。
- 外部系统：知脉、知微。

## 非范围

- 扩大管理者历史或当前组织作用域。
- 将人员+日期下合法多任务报工折叠为单条。
- 本需求不负责白名单扩围和回滚控制，详见 `ROL-001`。
