# WH-003 报工管理 - 报工查看与知脉同步/系统切换 - 业务规则说明书 (business-rules.md)

> **任务包编号**：WH-003  
> **任务包名称**：报工管理 - 报工查看与知脉同步/系统切换  

---

## 业务规则列表 (Business Rules)

- **`BR-141`**：**行方专属菜单控制**：非行方账号（如外包员工）在试图访问报工查看路由或 API 时，权限拦截器强制阻断返回 `40003`。
- **`BR-142`**：**知脉封板同步标识原则**：同步写回知脉的数据以知脉发送的封板标识 `workload_confirm_status` 为准，未封板的草稿数据不推送。
- **`BR-143`**：**双源优先互斥规则**：知脉工时核算模块在清洗同日工时记录时，知效数据优先级高于知微数据；`KEY(emp_no, work_date)` 一旦命中知效记录，知微记录直接丢弃。
- **`BR-144`**：**多任务明细冲突保留**：知效同一工号同日多条任务报工明细以打包 JSON 数组形式推送，明细冲突判定受待确认事项 `Q-07` 约束。
