# BASE-001 时效与基线 - 阶段时效与扣减/回退 - 需求说明书 (requirements.md)

> **任务包编号**：BASE-001  
> **任务包名称**：时效与基线 - 阶段时效与扣减/回退  
> **关联 PRD 章节**：7.1, 7.2  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-141`**：系统必须精准记录需求各个阶段的实际进入时间（enter_time）、离开时间（exit_time）及停留天数，计算公式为：`当前阶段时效 = 当前时间 - 进入该阶段时间`。
- **`FR-142`**：系统必须保留每个阶段的计划开始日期、计划结束日期、实际开始日期、实际结束日期，并支持计算各阶段的计划与实际时效偏差。
- **`FR-143`**：系统必须支持“时效扣减”功能，允许记录扣减天数（deduct_days）和扣减原因；业务时效与受理时效均保留“扣减前”与“扣减后”两个计算结果。
- **`FR-144`**：系统默认展示并对外提供**扣减后**的时效结果。
- **`FR-145`**：系统必须支持阶段回退时的时效累计规则，需求发生阶段回退时，重新进入该阶段的时效进行累计统计，禁止覆盖历史时效数据。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-015`**：阶段停留时长定时计算 Job J05 每日凌晨 03:00 执行，扣除工作日计算最新停留天数，全量统计表查询索引优化，单次大屏聚合响应小于 200ms。
