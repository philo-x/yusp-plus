# BASE-001 业务规则

| ID | 规则 |
|---|---|
| RULE-BASE-001 | 有效停留=阶段业务天数−审批通过扣减，且不得小于 0。 |
| BASE-001-RULE-001 | 未批准、被拒绝或撤回的扣减不得影响统计。 |
| BASE-001-RULE-002 | 扣减天数不得为负数或超过对应阶段累计可扣减天数。 |
| BASE-001-RULE-003 | 同一日期重复计算不得增加累计结果，查询计算不得写入重复事实。 |
| BASE-001-RULE-004 | 回退重入同一阶段时，每个区间独立保存并按批准日历累计。 |
| BASE-001-RULE-005 | 申请、审批、前后值、原因、操作者和时间必须审计。 |

## 异常与边界

- 无原因、负数、超额或无权扣减：拒绝且统计不变。
- 多次进入同一阶段：保留所有区间并累计。
- 扣减后小于 0：拒绝，不截断掩盖无效申请。
- 周末、节假日、半天假和跨时区：按 `TBD-BASE-001` 批准口径。

| 源 ID | 规范化边界 |
|---|---|
| ERR-BASE-001 | 未批准或超额扣减不改变有效天数，禁止产生负数或未审批生效。 |
