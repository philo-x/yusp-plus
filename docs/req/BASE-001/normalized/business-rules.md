# BASE-001 时效与基线 - 阶段时效与扣减/回退 - 业务规则说明书 (business-rules.md)

> **任务包编号**：BASE-001  
> **任务包名称**：时效与基线 - 阶段时效与扣减/回退  

---

## 业务规则列表 (Business Rules)

- **`BR-151`**：**双重结果保留原则**：系统计算业务时效与受理时效时，数据库字段 `raw_aging_days`（扣减前天数）与 `net_aging_days`（扣减后天数）必须同时更新存储；所有 UI 默认呈现 `net_aging_days`。
- **`BR-152`**：**回退不抹除历史原则**：需求多次进入同一阶段，`stage_aging_record` 表按事件追加写入，`total_stay_days = SUM(exit_time - enter_time)` 实现历史段连续累加。
- **`BR-153`**：**无需扩展审批流程**：原始 PRD 未定义扣减流程审批人，扣减天数在录入后直接更新结果，本版不人为增加审批拦截。
- **`BR-154`**：**上线终止累计**：需求进入“已上线”状态后，各阶段停留天数计算立即冻结；回退发生时解冻并继续计数。
