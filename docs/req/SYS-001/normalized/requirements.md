# SYS-001 非功能、安全与合规要求 - 需求说明书 (requirements.md)

> **任务包编号**：SYS-001  
> **任务包名称**：非功能、安全与合规要求  
> **关联 PRD 章节**：第 9 章 (9.1, 9.2, 9.3)  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-176`**：系统必须支持全量操作变更链路追踪，通过 `yusp-plus-utrace-core` 记录敏感字段变更与全量 SQL/API 操作日志（写至 `modify_trace_log` 表）。
- **`FR-177`**：系统必须满足国家与行内创新改造适配要求，框架支持 Java 17 / Spring Boot 3.4.5，中间件与数据库全面兼容 GoldenDB 6.x 73 张表结构。
- **`FR-178`**：系统必须保持现有监管结论一致：反洗钱监管要求不涉及、EAST 5.0 数据治理不涉及、数据安全脱敏规则按原PRD规定不自行新增规则。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-018`**：**性能要求**：系统必须按照现有生产用户规模，以最大同时在线用户数（5,000 在线，500 TPS）进行性能设计，全站 95% 页面加载与 REST API 响应时间小于 300ms。
- **`NFR-019`**：**可用性与高可用**：系统核心服务支持双活单体/微服务集群部署，数据库与 Redis 实现主备自动切换，系统可用性 `>= 99.99%`。
- **`NFR-020`**：**中间件防击穿防雪崩**：高频只读缓存设置随机 TTL 偏移，拦截非法 Key 采用布隆过滤器（`zx:bloom:req_code`），热点写操作全量使用 Redisson 分布式锁保护 GoldenDB。
- **`NFR-021`**：**接口安全性**：全量通信强制走 HTTPS / TLS 1.3，基于 OAuth2 (`yusp-plus-uaa-core`) 和 Bearer Token 验证身份，非法/超时请求一律拦截。
- **`NFR-022`**：**日志审计合规**：操作日志包含时间戳、操作人工号、IP 地址、API 路由、请求参数及变动前后的差异比较 JSON。
