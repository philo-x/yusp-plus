# ROL-001 灰度发布与回滚

> 来源场景：`SCN-OPS-002`
>
> 状态：DRAFT / 待联合评审

## 用户目标

按厂商、研发系统或部落逐批切换报工用户；达到严重故障门槛时恢复知微承接并保全知效期间数据。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| ROL-001 | Must | 白名单支持厂商、研发系统、部落三种粒度并记录生效时间、操作者和历史。 |
| ROL-002 | Must | 以人员+报工日期裁决唯一权威来源，保留该来源全部合法任务明细。 |
| ROL-003 | Must | 每次扩围前验证完整性、同步成功率、关键错误和反馈，保存指标门禁并由发布负责人批准。 |
| ROL-004 | Must | P0、工时大范围偏差或知脉同步中断达到批准阈值时停止扩围并决策回滚。 |
| ROL-005 | Must | 回滚关闭知效入口、恢复知微流量并导出知效期间数据；补偿不得覆盖未经确认或版本变化数据。 |
| ROL-006 | Must | 白名单、优先级和回滚状态只允许授权人员操作，未授权请求拒绝并记录。 |
| NFR-OBS-001 | Must | 灰度、回滚和 J05 产生指标、结构化日志、追踪号和可操作告警。 |
| NFR-OBS-002 | Must | 告警包含级别、对象、时间、影响、追踪号和处置入口，且不暴露敏感原文。 |

## 数据与作业

- 数据：`DATA-WH`、`DATA-SYNC`。
- 作业：`J05`。
- 外部系统：知效、知微、知脉。

## 非范围

- 无门禁证据或未批准直接扩围。
- 未核对即自动覆盖补偿数据。
- 普通用户修改白名单或回滚状态。
