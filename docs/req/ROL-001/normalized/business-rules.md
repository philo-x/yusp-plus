# ROL-001 业务规则

| ID | 规则 |
|---|---|
| RULE-ROL-001 | 每个人员+报工日期只有一个权威来源，采用来源的全部合法明细均保留。 |
| ROL-001-RULE-001 | 白名单规则版本化且不可篡改；未命中人员继续走知微。 |
| ROL-001-RULE-002 | 请求按受理时生效规则版本路由，规则变更不得追溯改写既有裁决。 |
| ROL-001-RULE-003 | 扩围门禁使用固定观测窗口和指标快照，结果必须可重算。 |
| ROL-001-RULE-004 | 达到冻结/回滚阈值立即停止新流量，不得继续扩围。 |
| ROL-001-RULE-005 | 回滚先恢复旧系统承接能力，再按裁决和补偿清单处置期间数据。 |
| ROL-001-RULE-006 | 版本或来源冲突不得自动覆盖，必须转人工审批。 |

## 异常与边界

- 同时命中多维白名单：按 `TBD-ROL-002` 批准优先级。
- P0、工时偏差或同步中断：按 `TBD-ROL-001` 阈值冻结/回滚。
- 旧系统回滚后不可用：升级应急处置，不得宣称回滚成功。
- 补偿冲突：暂停自动写入并形成完整人工清单。

| 源 ID | 规范化边界 |
|---|---|
| ERR-ROL-001 | 严重故障时停止扩围并按门槛回滚，禁止继续扩围或未经核对覆盖补偿。 |
| EDGE-ROL-001 | 同时命中多维白名单时按批准优先级和生效版本路由。 |
