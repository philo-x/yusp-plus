# MIG-001 业务规则

| ID | 规则 |
|---|---|
| MIG-001-RULE-001 | 迁移严格按组织/人员→需求/阶段→任务/协作→活跃报工→基线/考核排序。 |
| MIG-001-RULE-002 | 每批保存冻结点、增量水位、输入摘要、计数、检查点、操作者和时间。 |
| MIG-001-RULE-003 | 同一输入重跑不得创建重复业务对象或悬空关联。 |
| MIG-001-RULE-004 | 近 14 天活动报工包含第 14 日、不包含第 15 日；更早数据按批准方案汇总或归档。 |
| MIG-001-RULE-005 | 归档对象必须具有可检索定位和恢复证据，默认不进入在建查询。 |
| MIG-001-RULE-006 | 回滚不得删除源数据；版本冲突不得自动覆盖，须转人工。 |
| MIG-001-RULE-007 | `MIG-VAL-001～006` 全部通过并联合签署后方可切换。 |
| DATA-RULE-009 | 同步失败转人工；迁移可从明细、写前镜像和目标版本重算并按批次精确回滚。 |

## 执行门禁

1. 在隔离类生产环境用脱敏备份完成至少一次全流程演练。
2. 记录冻结点和最后增量水位，不依赖口头确认。
3. 全量校验和附加抽样全部完成。
4. 业务、数据、测试、运维共同签署对账报告。
5. 切换前完成回退演练并验证旧系统可继续承接。

## 异常与边界

- 异常记录未清零或关联校验失败：禁止切换，修复后重跑。
- 批次中断：从已确认检查点续传。
- 回滚目标版本变化：停止自动补偿并转人工。

| 源 ID | 规范化边界 |
|---|---|
| ERR-MIG-001 | 对账或关联失败阻止切换，禁止以抽样通过替代全量失败。 |
| EDGE-MIG-001 | 批次中断重跑从检查点继续，并保持业务对象唯一。 |
