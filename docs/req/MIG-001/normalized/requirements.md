# MIG-001 知微存量数据迁移

> 来源场景：`SCN-OPS-001`
>
> 状态：DRAFT / 待联合评审

## 用户目标

在不破坏知微源数据和考核连续性的前提下，将批准范围内数据迁入知效，并用全量校验证明一致性、可续传和可回滚。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| MIG-001 | Must | 迁移全量用户、组织和角色关系，建立知微工号与知效用户的唯一可追踪映射。 |
| MIG-002 | Must | 迁移全量在建需求、历史上线需求、阶段历史、日志、备忘录、历史与在建任务及协作关系。 |
| MIG-003 | Must | 切换时仅将近 14 天（含）活跃报工明细装入知效；更早历史考核汇总存入批准的数据仓库并可查询。 |
| MIG-004 | Must | 迁移全量历史基线、加塞、取消、调出和原计划快照，保持考核连续。 |
| MIG-005 | Must | 归档需求及规定范围历史报工进入批准的归档存储，并建立定位、摘要、保留期和恢复验证。 |
| MIG-006 | Must | 顺序为组织/人员→需求/阶段→任务/协作→活跃报工→基线/考核；失败批次可从检查点续传。 |
| MIG-007 | Must | 授权批次回滚不得删除源数据；更新/关联对象校验目标版本并按补偿序号倒序执行，冲突转人工。 |

## 接口、数据与执行

- 接口：`IF-095`～`IF-099`。
- 数据：`DATA-ORG/REQ/TASK/WH/BASE/SYNC`。
- 迁移记录：批次、输入摘要、检查点、对象明细、写前镜像、目标版本和补偿结果。

## 非范围

- 将全部历史报工明细装入知效活动库。
- 修改或删除知微源数据。
- 用抽样复核替代记录数、关键字段、关联和汇总的全量校验。
