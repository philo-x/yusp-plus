# TASK-002 安全任务同步与确认

> 来源场景：`SCN-TASK-002`
>
> 状态：DRAFT / 待联合评审

## 用户目标

知安安全要求随开发任务到达责任人，内容保持权威只读，开发确认可幂等回传并端到端追踪。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-TASK-006 | Must | 安全类任务由知安同步，知效不得修改安全要求原文。 |
| FR-TASK-008 | Must | 安全任务仅在关联开发任务且满足批准报工条件时计工时。 |
| FR-TASK-012 | Must | 涉及安全要求的开发任务醒目标识，并展示知安同步的场景、需求和设计要点。 |
| FR-TASK-013 | Must | 开发责任人显式确认，结果幂等回传知安；未确认不得标记安全要求完成。 |
| FR-CROSS-001 | Must | 缺标识、无法映射、状态非法或载荷失败必须形成可处理异常。 |
| FR-CROSS-002 | Must | 同步与确认使用幂等键，有限重试耗尽后转人工。 |

## 接口、数据与作业

- 接口：`IF-037`、`IF-038`、`IF-051`、`IF-108`。
- 数据：`DATA-TASK`、`DATA-SYNC`。
- 作业：`J12`。
- 外部系统：知安。

## 非范围

- 修改或重写知安安全要求原文。
- 在判定规则批准前自行决定安全任务计工资格。
- 将外部回传失败伪装为确认成功。
