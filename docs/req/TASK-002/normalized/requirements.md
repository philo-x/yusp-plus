# TASK-002 任务管理 - 任务分发、认领与流转 - 需求说明书 (requirements.md)

> **任务包编号**：TASK-002  
> **任务包名称**：任务管理 - 任务分发、认领与流转  
> **关联 PRD 章节**：5.6, 5.8  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-091`**：系统必须支持任务的分发、主动认领和负责人流转，支持将任务指定给单人或多人分配。
- **`FR-092`**：系统必须支持未分配或无人认领任务的隔离管理，将其汇总展示在“未分配任务”独立区域，供有权限的团队领导或 PM 进行后续分发与认领。
- **`FR-093`**：系统必须提供嵌入式表格在线编辑能力，允许有权限的用户在任务列表中直接对责任人、状态、计划时间等可编辑字段进行行内编辑。
- **`FR-094`**：系统必须在任务状态或计划时间保存成功后，自动触发对应的下游业务流程（如状态同步至知脉或触发流程审批）。
- **`FR-095`**：系统必须提供待办任务列表和已办结任务列表，待办列表展示已分配未完成及未分配任务，已办结列表展示已完成任务；待办列表数据与个人工作台同源。
- **`FR-096`**：系统必须支持从宏观指标层级连击下钻至组织架构、OKR、项目及具体任务明细。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-010`**：嵌入式表格行内编辑保存时的响应时间不超过 150ms，且编辑保存时自动上分布式锁防重复提交。
