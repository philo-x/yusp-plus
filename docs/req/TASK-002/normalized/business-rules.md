# TASK-002 业务规则

| ID | 规则 |
|---|---|
| TASK-002-RULE-001 | 同一租户+知安源任务标识唯一；事件按事件 ID 和源版本幂等。 |
| TASK-002-RULE-002 | 安全场景、需求和设计要点为只读权威内容，版本更新须保留历史。 |
| TASK-002-RULE-003 | 只有任务开发责任人或获批准角色可确认，确认人和时间必须记录。 |
| TASK-002-RULE-004 | 首次确认形成一次业务结果；重复提交或回传重试不得重复确认。 |
| TASK-002-RULE-005 | 安全任务是否计工由 `TBD-TASK-001` 的批准判定表决定。 |
| TASK-002-RULE-006 | 无法映射和外部暂时失败不得丢弃；有限重试后进入人工处置。 |

## 异常与边界

- 无法映射需求/开发任务：进入同步异常，不创建孤立正常任务。
- 普通用户编辑原文：拒绝并记录安全事件。
- 非责任人确认：按服务端权限拒绝。
- 知安暂时不可用：保留待重试状态，不伪造成功。
- 重复或版本更新事件：不重复创建，旧版本可追溯。
