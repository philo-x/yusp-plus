## 验收标准列表 (Acceptance Criteria)

- **`AC-116`**：**需求类任务建删拦截测试**：在需求类任务界面无“新建”按钮；尝试通过 API `POST /api/task/create` 创建需求类任务，系统返回 `retCode: 40010` 拦截。
- **`AC-117`**：**个人子任务弹窗校验测试**：点击“认领需求任务”，系统自动弹出子任务新增窗口；清空“估算工作量”点击提交，校验提示“工作量为必填项”。
- **`AC-118`**：**双向人员展示校验**：打开开发任务 D-01，界面“测试人员”栏展示对应系统 SIT 测试任务的测试人员张三；打开测试任务 T-01，“开发人员”栏展示李四。
- **`AC-119`**：**事务类任务批量分发测试**：勾选 3 名团队成员下发“部门例会”事务任务，任务表新增 3 条记录，3 名成员的个人待办列表中各自出现该任务。
- **`AC-120`**：**安全内容确认与知安回传测试**：开发人员在任务卡片勾选“安全内容已开发完成”并保存，安全确认状态更新，后端成功调用知安 API 回传确认结果。
