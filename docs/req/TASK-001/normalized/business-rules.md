# TASK-001 业务规则

| ID | 规则 |
|---|---|
| RULE-TASK-001 | 需求类任务由知脉主控；事务类任务不计工时。 |
| RULE-TASK-002 | 个人子任务“进行中”即点亮，离开即取消；每人每天最多 5 个。 |
| TASK-001-RULE-001 | 开发/SIT 子任务必须有且只有一个责任人，并具有有效工作量。 |
| TASK-001-RULE-002 | 任务类型决定创建、删除、点亮和计工权限，不得通过接口绕过类型限制。 |
| TASK-001-RULE-003 | 负责人、计划和状态每次变化均保留历史，不得覆盖既有操作事实。 |
| TASK-001-RULE-004 | 每人每日每任务仅有一条点亮事实；J03 使用历史最终态而不是当前任务状态反推。 |
| TASK-001-RULE-005 | 模板实例是独立任务树，后续修改不得回写模板。 |
| DATA-RULE-005 | 每人每日每任务仅一条点亮事实，J03 读取前一日最终历史态，每日最多 5 个。 |

## 异常与边界

- 缺责任人或工作量：分配失败且不创建不完整子任务。
- 无权流转或非法状态：拒绝并保持原状态。
- 第 6 个点亮：拒绝本次操作，原 5 个点亮和状态不变。
- 批量混合合法/非法数据：合法项按事务策略处理，失败项不越权；返回准确逐项结果。

| 源 ID | 规范化边界 |
|---|---|
| ERR-BATCH-001 | 批量部分非法时返回准确汇总和逐项原因，不以整体成功掩盖失败。 |
| EDGE-TASK-001 | 第 6 个点亮操作被拒绝，原 5 个任务状态不变。 |
