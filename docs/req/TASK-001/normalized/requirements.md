# TASK-001 任务分配、认领与推进

> 来源场景：`SCN-TASK-001`
>
> 状态：DRAFT / 待联合评审

## 用户目标

将需求或项目工作分配给明确责任人，通过任务类型、状态、子任务和历史反映实际执行情况。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-TASK-001 | Must | 需求类任务由知脉同步；知效不得新建/删除源任务，但可分发、认领、流转负责人和创建子任务。 |
| FR-TASK-002 | Must | 开发和 SIT 测试任务分配后创建个人子任务，唯一责任人和工作量必填，计划默认继承需求排期且调整留痕。 |
| FR-TASK-003 | Must | 对应系统的开发和 SIT 测试任务互相展示已分配人员。 |
| FR-TASK-004 | Must | 项目类任务由项目经理或团队长创建，支持批准模板的 Excel 导入导出。 |
| FR-TASK-005 | Must | 事务类任务允许全员创建，但不得点亮或计工时。 |
| FR-TASK-006 | Must | 安全类任务由知安同步且原文不可在知效修改。 |
| FR-TASK-007 | Must | 至少支持未开始、进行中、阻塞、暂停、已完成，状态变化记录操作者、时间、前后值和适用原因。 |
| FR-TASK-008 | Must | 需求类、项目类可计工时；安全类仅满足批准关联条件时可计；事务类永不计工时。 |
| FR-TASK-009 | Must | 个人子任务进入“进行中”即当日点亮，离开即取消。 |
| FR-TASK-010 | Must | 每人每天最多同时点亮 5 个可报工任务，第 6 个必须拒绝且不改变原状态。 |
| FR-TASK-011 | Must | 支持行内编辑和批量分发/更新，批量请求逐项校验并返回逐项结果。 |
| FR-TASK-014 | Should | 支持任务模板创建、查询和应用，实例化不得修改模板。 |
| FR-CROSS-003 | Must | 批量操作返回总数、成功数、失败数和逐项失败原因。 |

## 接口与数据

- 接口：`IF-028`～`IF-036`、`IF-039`～`IF-050`。
- 数据：`DATA-TASK`、`DATA-ORG`。
- 外部依赖：知脉任务主数据。

## 非范围

- 在知效新建或删除知脉源任务。
- 允许事务类任务进入点亮或报工候选。
- 本需求不负责知安内容确认流程，详见 `TASK-002`。
