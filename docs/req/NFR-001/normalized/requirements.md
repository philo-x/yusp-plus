# NFR-001 平台质量、安全与运行保障

> 跨场景质量需求包
>
> 状态：DRAFT / 待联合评审

## 目标

为所有业务需求建立统一的性能、容量、安全、隐私、审计、可用性、容灾、兼容性、可维护性、可观测性和易用性门槛。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| NFR-PERF-001 | Must | 批准环境和数据量下，需求看板/列表响应≤1.5 秒，甘特图响应/渲染≤2.0 秒；统计口径待批准。 |
| NFR-PERF-002 | Must | 报工高峰写入目标 2,000 TPS，且不得丢失已确认请求；负载模型待批准。 |
| NFR-CAP-001 | Must | 支持需求 50 万/年、任务 50 万/年、报工 25 万/年，三年估算≤200GB，并保留至少 300GB 规划空间。 |
| NFR-SEC-001 | Must | 使用知悉 OAuth2，不维护独立硬编码密码。 |
| NFR-SEC-002 | Must | 服务端对每个接口执行功能和数据作用域授权，遵循最小权限。 |
| NFR-SEC-003 | Must | 敏感数据传输加密；静态加密范围、算法和密钥托管按批准方案。 |
| NFR-SEC-004 | Must | 输入、文件、模板和富文本执行服务端安全校验，防止注入、越权、恶意文件和脚本。 |
| NFR-SEC-005 | Must | 上线前完成依赖、代码、配置和渗透测试，高危/严重问题关闭或正式接受。 |
| NFR-SEC-006 | Must | 附件校验大小、扩展名、MIME/特征并执行批准的恶意内容检测。 |
| NFR-PRIV-001 | Must | 手机号、身份证号、邮箱在非必要场景脱敏，导出和日志不得扩大暴露。 |
| NFR-AUD-001 | Must | 工时、基线、扣减、权限、同步、迁移和回滚记录主体、IP、时间、对象、前后值、原因、结果和追踪号。 |
| NFR-AUD-002 | Must | 审计日志保存不少于 3 年。 |
| NFR-AVL-001 | Must | 上线前批准可用率目标、维护扣除和测量点，并由监控按公式计算。 |
| NFR-DR-001 | Must | 上线前批准备份范围、频率、保留期、RPO/RTO，并完成恢复演练。 |
| NFR-COMP-001 | Must | 在批准的 Java 17、GoldenDB 6.x、TongWeb、Vue 2 环境通过功能、性能和兼容测试。 |
| NFR-MAINT-001 | Must | 配置、字典、调度和外部端点不得硬编码，须按环境管理且变更可审计。 |
| NFR-OBS-001 | Must | 核心接口、J01～J13、同步、迁移和灰度产生指标、结构化日志、追踪号和可操作告警。 |
| NFR-OBS-002 | Must | 告警包含级别、对象、时间、影响、追踪号和处置入口，且不暴露敏感原文。 |
| NFR-USE-001 | Should | 核心列表、表单和状态操作提供明确标签、校验提示和键盘可达性。 |

## 适用范围

本包适用于 `AUTH-001`、`REQ-*`、`TASK-*`、`WH-*`、`BASE-*`、`MIG-001` 和 `ROL-001` 的所有接口、数据、调度及部署环境。
