# NFR-001 验收标准

## AC-NFR-001：性能与容量

- 在批准统计口径、预热、持续时长、错误率和数据量下，以 500 并发验证看板/列表≤1.5 秒、甘特图≤2.0 秒。
- 在批准负载模型下验证报工写入达到 2,000 TPS 且无已确认请求丢失。
- 三年容量模型覆盖索引、日志、附件和增长假设，并达到规定业务量与规划空间。

## AC-NFR-002：安全、隐私与审计

- 登录生命周期和角色×对象允许/拒绝矩阵通过，代码/配置无硬编码凭据。
- TLS、静态加密、输入/富文本/附件、依赖/代码/配置扫描和渗透测试通过。
- 页面、接口、导出、日志和告警符合敏感字段脱敏矩阵。
- 审计字段完整、不可篡改，生命周期与恢复抽查满足不少于 3 年。

## AC-NFR-003：可用性、恢复、兼容和可观测

- 监控按批准公式输出可用率；恢复演练在批准 RPO/RTO 内完成并通过数据校验。
- 目标 Java/数据库/中间件/前端环境无阻断兼容缺陷。
- 配置可按环境管理，业务代码无硬编码。
- 对接口、J01～J13、同步、迁移和灰度注入成功、业务失败、超时、积压，均可由指标、日志、追踪和告警定位。
- 核心旅程按批准易用性标准完成走查。

## AC-E2E-001：跨需求端到端门禁

- 从知脉需求进入开始，依次覆盖认证授权、需求视图、阶段流转、任务/安全、点亮/考勤/报工、知脉写回、时效扣减、月度基线和 90 日归档。
- 同时覆盖迁移、灰度、回滚、重复事件、权限拒绝、外部依赖失败和重试耗尽。
- 11 个核心场景及全部 Must 需求均有证据，且不存在未接受的阻断缺陷。

- 无未关闭或未正式接受的严重/高危安全缺陷。
