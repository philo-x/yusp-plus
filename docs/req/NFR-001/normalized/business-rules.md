# NFR-001 业务规则

| ID | 规则 |
|---|---|
| RULE-AUD-001 | 工时、基线、扣减、权限、同步、迁移、灰度和回滚必须审计。 |
| NFR-001-RULE-001 | 认证、授权、租户和数据作用域在服务端执行，默认拒绝。 |
| NFR-001-RULE-002 | 敏感值不得出现在日志、错误、告警或非必要导出中。 |
| NFR-001-RULE-003 | 所有外部输入均校验；业务校验错误不得自动重试。 |
| NFR-001-RULE-004 | 仅明确可重试的临时错误执行有限重试，耗尽后可观测并转人工。 |
| NFR-001-RULE-005 | 审计、事件、阶段、迁移、基线和灰度证据只追加，普通用户不可修改删除。 |
| NFR-001-RULE-006 | 配置变更按环境隔离并审计，凭据和密钥不得硬编码或提交仓库。 |
| NFR-001-RULE-007 | 性能结果必须来自批准负载模型，不能用年度均值证明峰值 TPS。 |
| DATA-RULE-010 | 审计、事件、阶段、迁移、基线、灰度和归档记录只追加，总保存不少于 3 年且可恢复检索。 |

## 接口统一规则

- TLS、UTF-8、带时区 ISO 8601 和 `YYYY-MM-DD` 业务日期。
- 统一追踪号、业务状态、错误码和安全错误消息。
- 创建、同步、迁移、批量和调度入口使用业务幂等键。
- 更新使用版本号或等价并发控制，冲突不得静默覆盖。
- 批量结果明确总数、成功数、失败数和逐项原因。
