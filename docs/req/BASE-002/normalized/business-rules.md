# BASE-002 业务规则

| ID | 规则 |
|---|---|
| RULE-BASE-002 | 5 日生成/取消基线，6—30 日识别加塞和调出，所有作业幂等。 |
| RULE-BASE-003 | 原计划只在首次为空时保存；取消只删除当月录入值。 |
| BASE-002-RULE-001 | 同一需求同一基线月只有一个当前结果，同时保留状态历史和规则版本。 |
| BASE-002-RULE-002 | 基线调出保留原基线记录和计划快照，不作物理删除。 |
| BASE-002-RULE-003 | 5/6 日、月末和跨年边界按业务时区及目标计划完成月份判定。 |
| BASE-002-RULE-004 | 首次计划上线日期一旦保存，不得被后续计划日期覆盖。 |
| BASE-002-RULE-005 | 无计划日期或配置不完整的需求进入待处理清单，不得自行填充。 |
| DATA-RULE-008 | 扣减仅批准后生效；基线状态同月唯一并保留历史、规则版本和计划快照。 |

## 异常与边界

- 重复作业：不创建重复记录或重复状态变化。
- 跨月已有原计划：新基线/加塞不得覆盖，取消不得删除。
- 5 日与 6 日边界：严格按批准业务时区。
- 历史快照人工修改：按权限拒绝并审计。

| 源 ID | 规范化边界 |
|---|---|
| EDGE-BASE-001 | 月末、跨年及 5/6 日边界使用业务时区和目标计划月份。 |
| EDGE-BASE-002 | 跨月原计划不得被新基线覆盖或被取消删除。 |
