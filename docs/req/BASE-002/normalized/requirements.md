# BASE-002 月度基线、加塞与计划变化

> 来源场景：`SCN-BASE-002`
>
> 状态：DRAFT / 待联合评审

## 用户目标

形成稳定的月度交付考核快照，识别基线取消、加塞和调出，并保留首次计划及后续变化。

## 规范性需求

| ID | 优先级 | 需求 |
|---|---|---|
| FR-BASE-005 | Must | J07 每月 5 日 00:00、12:00、16:00、22:00 幂等生成当月基线。 |
| FR-BASE-006 | Must | J08 每月 5 日 01:00、14:00、18:00、23:00 取消计划月已变为非当月的基线并留痕。 |
| FR-BASE-007 | Must | J09 每月 6—30 日四次将未入基线但计划完成月为当月的需求标记为加塞。 |
| FR-BASE-008 | Must | 5 日后已入基线但计划完成月改为非当月的需求标记调出并保留原基线。 |
| FR-BASE-009 | Must | 首次纳入基线或加塞且原计划为空时保存原计划，已有值不得覆盖。 |
| FR-BASE-010 | Must | 基线/加塞取消只删除当前月录入的原计划，跨月值保留。 |
| FR-BASE-011 | Must | 保存首次计划上线日期，后续变化区分计划偏差和计划变更并可还原。 |

## 接口、数据与作业

- 接口：`IF-071`～`IF-081`。
- 数据：`DATA-REQ`、`DATA-BASE`。
- 作业：`J07`、`J08`、`J09`。

## 非范围

- 修改历史基线快照或覆盖原计划。
- 由实现自行扩展 30 日后至次月 5 日前的处理规则。
