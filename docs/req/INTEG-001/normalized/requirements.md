# INTEG-001 外部系统集成与数据迁移 - 需求说明书 (requirements.md)

> **任务包编号**：INTEG-001  
> **任务包名称**：外部系统集成与数据迁移  
> **关联 PRD 章节**：第 8 章 (8.1, 8.2, 8.3)  

---

## 1. 功能需求 (Functional Requirements)

- **`FR-166`**：系统必须建立统一外部系统集成层，明确对接 6 大外部系统/模块的核心职责与通信契约：
  1. **知脉**：需求提出同步、分析/排期节点、开发任务同步、需求计划同步、工作量确认查询、报工数据核算写回。
  2. **主系统**：DEV 准出事件、SIT 准入事件推送。
  3. **知测**：最后一次 SIT 准出、第一次 UAT 准入、最后一次 UAT 准出事件推送。
  4. **知安**：安全场景/需求/要点同步及开发人员确认结果回传。
  5. **钉钉**：外包人员打卡与请假考勤时长每日同步。
  6. **上线或变更流程**：运维通过结果及上线日期同步。
- **`FR-167`**：系统必须支持存量知微系统数据的平滑迁移，迁移范围至少覆盖知微系统存量考核数据及关联历史记录。
- **`FR-168`**：数据迁移实施前必须形成专项数据迁移方案，梳理历史数据范围并编写 Python/SQL 批处理迁移校验脚本，确保过程连续性。
- **`FR-169`**：系统必须完成前端原型设计与交互覆盖，主页、个人工作台、需求管理视图、任务列表及报工管理页面原型必须 100% 覆盖 PRD 已明确的字段、权限与业务规则。

---

## 2. 非功能需求 (Non-Functional Requirements)

- **`NFR-017`**：外部系统回调通用防重与降级重试机制（Job J01, J10, J11, J12），所有外部接口在 5xx 或 Timeout 异常时自动写入 `sync_conflict` 表并退避重试 3 次。
