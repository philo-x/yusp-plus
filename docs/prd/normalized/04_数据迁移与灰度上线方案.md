### 1.2 全量一致性校验

| ID | 必检项目 | 通过条件 | 证据 |
|---|---|---|---|
| MIG-VAL-001 | 对象记录数 | 每个迁移范围的源数=成功数+经业务批准的排除数+异常数；异常数必须为0方可切换 | 批次计数报告 |
| MIG-VAL-002 | 关键字段 | 所有迁移记录的业务标识、状态、日期、责任人、组织、分钟数和基线状态逐条一致 | 全量字段比对报告 |
| MIG-VAL-003 | 关联完整性 | 需求—阶段、需求—任务、人员—组织、人员—报工、任务—报工不存在非预期悬空引用 | 引用完整性报告 |
| MIG-VAL-004 | 业务汇总 | 按月、组织、人员、需求汇总的工时、需求数和基线数迁移前后一致 | 汇总对账报告 |
| MIG-VAL-005 | 抽样复核 | 至少抽样10%记录进行人工/独立复核，作为全量自动校验的附加检查 | 抽样清单及签字 |
| MIG-VAL-006 | 可重复性 | 同一输入重跑不新增重复业务对象，校验结果稳定 | 重跑测试报告 |

抽样不得替代MIG-VAL-001～004的全量校验，也不得以抽样通过宣称“零数据丢失”。

### 1.3 迁移执行门禁

1. 预迁移：在隔离的类生产环境使用脱敏备份完成至少一次全流程演练。
2. 冻结与增量：记录源系统冻结点和最后增量水位，不得依赖人工口头确认。
3. 装载：每批记录批次号、输入摘要、成功/失败数、开始/结束时间和操作者。
4. 校验：MIG-VAL-001～006全部通过后方可申请切换。
5. 签署：业务、数据、测试和运维共同签署对账报告。
6. 回退：切换前必须完成一次回退演练并验证旧系统可继续承接业务。
