易子航：报工记录 CRUD 接口（约2周）
任务： 实现外包人员"我的报工"的个人增删改查功能

接口清单（4个）：

POST /api/work-hours/create - 新增工时记录
PUT /api/work-hours/{id} - 修改工时记录
DELETE /api/work-hours/{id} - 删除工时记录
GET /api/work-hours/my - 个人报工记录查询（列表「支持过滤条件」）
涉及业务规则：

14天填报窗口期（超过不可增删改）
已确认工作量锁死不可修改
每日报工总时长 ≤ 当日考勤总时长
只能选择已分配的开发任务

难易度： 中等偏下。典型的单表CRUD + 几个业务校验逻辑，不涉及多系统集成。

陈子俊：考勤同步/报工数据校验/配置接口（约2周）
任务： 实现报工前端的合法性校验和后端的配置参数管理

接口清单（3个）：
GET /api/work-hours/sync/work-time -考勤数据同步定时任务(14天增量的考勤数据)
GET /api/work-hours/validate - 工时合法性校验（前端填报时实时校验）
GET /api/work-hours/config - 获取报工配置参数
涉及业务规则：

校验日期是否在14天窗口内。
校验报工人的厂商和开发任务厂商是否一致。
校验当日累计总时长 ≤ 考勤总时长
检验点亮卡片数据是否超过每日最大点亮任务数
配置参数（可修改天数、每日最大点亮任务数等）从数据字典读取，支持管理员动态调整


涉及外部集成： 知脉

难易度： 中等偏下。主要是校验逻辑的编写和配置参数的读取，业务独立性强，不会影响其他功能。

1.attendance_sync_record 考勤同步记录表
2.work_hour_config 报工配置表（14天，每日最大点亮任务数）

王亮：管理端报工查看与导出（约2周）
任务： 实现管理端的分级查看报工数据和导出功能。


接口清单（3个）：

GET /api/work-hours/admin/list - 管理端报工查看（按权限分级过滤列表）
GET /api/work-hours/admin/export - 报工数据导出Excel
GET /api/work-hours/{id}/history - 查看任意报工记录的修改历史
涉及业务规则：

权限分级：行方负责人→中心经理→小队长→PMO/领导
多维度查询：日期、姓名、工号、公司、需求、任务、系统、是否确认工作量
导出数量 ≤ 1万条，流式写入Excel

难易度： 中等偏下。主要是多表关联查询 + 权限过滤 + Excel导出。业务完整，做完能看到完整的管理端功能。
